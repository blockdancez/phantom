# 验证：用实际运行证明测试通过

你刚刚进行了一轮测试编写。现在请通过**实际运行**验证。

## 第一步：运行所有测试

```bash
# 根据项目类型执行（选择适用的）
# Node.js: npm test
# Python: pytest -v
# Go: go test ./...
# Playwright: npx playwright test
```

完整输出测试结果。

## 第二步：分析测试结果

- 有多少测试？多少通过？多少失败？
- 如果有测试失败 → 分析原因，修复**代码**（不是修复测试来适配错误代码），然后重新运行
- 如果全部通过 → 继续下一步

## 第三步：检查测试覆盖

对照需求文档中的每个功能点：
- ✅ 有对应测试且通过
- ❌ 缺少测试

如果有 ❌ → 补充测试并运行。

## 第四步：集成验证（如适用）

如果项目有前后端或多个服务：
1. 启动所有服务
2. 运行集成测试或手动 curl 验证关键流程
3. 关闭服务

## 第五步：结论

如果：
- 所有测试通过（0 failures）
- 核心功能都有测试覆盖
- 集成验证通过（如适用）

输出：PHASE_COMPLETE

否则，修复后**从第一步重新运行**。
