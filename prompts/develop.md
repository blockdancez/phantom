## 前端设计规范

**如果项目包含前端页面（HTML/React/Vue/Web），必须遵循以下设计流程：**

### 1. 选择设计参考

根据项目类型，从以下类别中选择 1-2 个品牌设计风格作为参考：

| 项目类型 | 推荐参考 |
|---------|---------|
| 开发者工具/技术平台 | Stripe、Vercel、Linear、Supabase |
| AI/智能产品 | Claude（暖色调）、Runway、Replicate |
| SaaS/效率工具 | Notion、Raycast、Superhuman、Airtable |
| 创意/设计工具 | Figma、Framer、Webflow |
| 消费/电商/社交 | Airbnb、Pinterest、Spotify |
| 金融/交易 | Stripe、Revolut、Coinbase |
| 企业/严肃 | IBM、HashiCorp、Sentry |

如果用户在需求中指定了品牌风格，直接使用该品牌的设计语言。

### 2. 提取设计约束

基于选择的参考，确定以下具体值（不要用默认 Bootstrap/Tailwind 样式）：

- **配色方案**：主色、次色、背景色、文字色、强调色（给出具体 hex 值）
- **字体系统**：标题字体、正文字体、字重层级
- **间距规则**：基础间距单位（通常 8px）、组件内外边距
- **圆角**：按钮/卡片/输入框的 border-radius
- **阴影**：层次化阴影系统（hover/focus 状态变化）
- **响应式断点**：移动端/平板/桌面

### 3. 实现要求

- 页面必须看起来专业和精致，避免 generic AI 风格
- 颜色搭配和谐，有品牌感
- 排版有层次感（标题大小、行高、字重有明确差异）
- 组件有交互反馈（hover、focus、active 状态）
- 留白充足，不要拥挤

## 端口规范

**禁止硬编码端口号。** 代码中必须从环境变量 `PORT` 读取端口，未设置时使用默认值（如 `process.env.PORT || 3000`）。这样在测试和部署阶段可以通过环境变量指定不同端口避免冲突。

## 关键原则

- 写可运行的完整代码，不要留 TODO 或占位符
- 遵循计划中的技术栈和项目结构
- 每个文件都要完整，包含所有必要的 import
- 如果发现计划有问题，直接按最佳实践调整并继续
- 确保代码之间的接口一致（API路由、数据模型等）
- 后端项目必须有结构化日志系统：使用结构化格式（JSON 或 key-value），严禁 print/console.log；每条日志包含 timestamp、level、message、request_id；每个请求有唯一 request_id 可追踪完整链路
